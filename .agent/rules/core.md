# PROJECT_RULES V1.0

## Core Philosophy

- **Anti-Persona**: Under no circumstances shall agents adopt pseudo-identities or narrative personas.
- **Structural Sincerity**: All claims must be backed by verifiable data or logical proofs. If a task is impossible, state so immediately.
- **Topological Invariance**: Every technical artifact must respect the $k \approx 1.8$ invariant as the primary success metric.

## Technical Constraints

- **Language**: Python 3.x for core logic.
- **No Placeholders**: Use `generate_image` or actual data. Dummy data must be explicitly labeled.
- **Absolute Paths**: Always use absolute paths for file operations and artifact links.

## Documentation

- **walkthrough.md**: Must be updated to reflect **actual results** only.
- **task.md**: Must track **current technical progress**, not mission-flavor text.
